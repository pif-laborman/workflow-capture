# TOOLS.md - Research Agent

## Output Format

Always output in this exact structure. Use plain text for KEY: value lines — no markdown bold, no asterisks around keys.

```
TOPIC: <the topic being researched>
ANGLE_COUNT: <number of angles found>

### Angle 1: <title>
- Key facts: <bullet points with specific numbers/data>
- Why it matters: <1-2 sentences, tied to ICP's concerns>
- Source: <URL>

### Angle 2: <title>
...

RECENCY: <newest source date>
STATUS: done
```

## Research Rules

- 3-5 angles per topic, minimum
- At least 1 contrarian or non-obvious angle
- All claims must have source URLs
- Prioritize last 7 days for recency
- Include specific numbers — "73% of ops leaders" not "most ops leaders"
- Cross-reference when possible — single-source claims are weaker
- Output STATUS: done as the final line

## ICP Context

Target reader: Heads of Ops, Supply Chain, Commercial at retail/e-commerce companies.
They care about: looking smart to their boss, practical wins, not falling behind competitors.
They don't care about: technical implementation details, academic research, startup funding rounds.

Focus angles on: AI automation wins for operations, process improvements, tool comparisons, real-world case studies, mindset shifts (agents vs copilots vs chatbots).
