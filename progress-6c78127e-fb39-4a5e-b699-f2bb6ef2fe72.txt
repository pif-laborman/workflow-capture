## Codebase Patterns
- Next.js 14 App Router, TypeScript, no Tailwind, no UI library
- Hooks go in `lib/hooks/` directory, prefixed with `use`
- Tests: vitest + @testing-library/react, jsdom env, run with `npm test`
- Test files live in `app/__tests__/`
- Vitest config: vitest.config.ts with @vitejs/plugin-react and `@/` path alias
- CSS: plain CSS in app/globals.css with Rekon design tokens as CSS custom properties
- Fonts: Google Fonts loaded via <link> in layout.tsx (DM Sans, Inter, Fragment Mono)
- Shared button classes: .btn-primary, .btn-outline, .btn-stop in globals.css
- Shared card class: .card in globals.css
- Shared types in `lib/types.ts`, constants in `lib/constants.ts`
- System prompts in `lib/prompts/observe.ts` and `lib/prompts/finalize.ts`
- Cooldown state in `lib/cooldown.ts` (separate module so route files stay clean for Next.js)
- Keyframe selection in `lib/keyframes.ts` (separate module, same pattern)
- Next.js route files cannot export non-route functions (GET, POST, etc.) — use separate modules for testable state
- Streaming API routes: use `client.messages.stream()` with `.on('text', cb)` + `await .finalMessage()`, return `new Response(readableStream)`
- App state machine: lib/state.ts (AppStateContext + useAppState hook), components/AppShell.tsx (top-level client component)
- Components go in `components/` dir at project root
- Testing-library tests need explicit `cleanup()` in `afterEach` to avoid DOM accumulation across tests
- AppStateContext includes selectedWorkflowId/setSelectedWorkflowId for navigating to saved workflow results
- localStorage key for saved workflows: 'workflow-capture-sessions' (SavedWorkflow[])
